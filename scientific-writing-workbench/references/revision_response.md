# Revision Response Workflow

Use this reference when responding to reviewers, supervisors, or internal coauthors.

## Response principles

- Answer the comment directly before defending the manuscript.
- Make the manuscript change explicit.
- If you disagree, explain the reasoning respectfully and cite evidence or policy where possible.
- Avoid claiming a change was made unless the exact location is identified.

## Response table

Track revisions in a table or structured list with:

| comment_id | reviewer_comment | response | manuscript_change | changed_location | status |
| --- | --- | --- | --- | --- | --- |
| R1 | Example comment | Example response | Clarified cohort selection | Methods, paragraph 2 | done |

Statuses can be `done`, `partially_done`, `not_done`, or `needs_author_input`.

## Suggested workflow

1. Group comments by theme or section.
2. Separate requests for clarification from requests for new analysis or new experiments.
3. Draft the response note and the manuscript edit together.
4. Update the evidence map if the comment affects claims or citations.
5. Verify every quoted section location after revisions are applied.

## Tone guidance

- Thank the reviewer once in the opening, not in every line item.
- Be specific and calm.
- Avoid defensive filler.
- Keep response letters auditable.

## Closing check

Before finalizing a response package, verify that:

- every reviewer point has a matching response
- every claimed manuscript edit has a location
- new citations added during revision have passed citation validation
